# Source: SBU Official Course Evaluation — ESE 124, Professor Alexa Doboli

## What was valuable about this course

The videos that Professor Doboli posts before the labs are extremely helpful as he works through how to complete the problems from the lab. The TAs are a good resource to use when you get stuck as there are a ton of office hours and many have taken the exact same course before. The project at the end of the course although difficult really helps with understanding how to use everything we learned throughout the semester since there is no right way to complete it.

---

The example code he writes is extremely crucial for how you should be doing the lab, homeworks, and exam problems. Whenever he writes code down you should definitely copy it down as you will find yourself copy and pasting that code a lot and refactoring it. The professor did state in the beginning of the semester that you should be pestering TAs on every single lab question to get a good grade.

---

I like the fact that we have a lab section where you can ask any coding questions to the TAs, especially considering the fact that there are many nuances with the C programming language. The midterm and final were also very fair, which is a great improvement from previous years.

---

The most valuable part of this course is that you learn a lot of basics of C programming and understand the software tools.

Fundamentals of C Programming.

## What could be improved about this course

While I liked Doboli's teaching style, it would benefit students greatly if he included supplementary notes to look over in your own time, mainly due to the fact that the C language is broad and it may be hard to find information online. Make sure that as a student you ask any questions about the lab exercises, as you will lose a lot of points if you do it wrong.

---

Clearer questions on the lab. More test cases for each coding exercise. This class is intended for complete beginners with no coding experience. However, from the way he covered the content and the difficulty of the tests, I would have borderline failed this course if I came in with no coding experience. ChatGPT, YouTube, and your TAs are your best friends for this course.

---

Something that everyone would benefit from would be further instruction on how to use a queue or stack for strings and how to properly use ifdefine. More importantly, commented code for the more difficult lab exercises after labs are due would be a helpful resource for students seeking to improve their understanding.

---

The course can be improved if the labs are group work instead of individual exercises. In addition, the limitation of the functions and methods for each question should be listed on the test paper, so students don't need to spend time asking the professor and TA about each question on the exam.

---

The lab and project descriptions felt extremely vague.

---

Lab activities and projects are at times unclear.
